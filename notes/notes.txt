007: route pages are defined by src/pages/ and the file name
(src/pages -- is required)
- src/pages/index.astro -> /
- src/pages/about.astro -> /about
- src/pages/blog/post.astro -> /blog/post
- src/pages/[slug].astro -> /:slug (dynamic route)
- src/pages/[...catchall].astro -> /* (catch-all route)

Astro will create a page for each .astro file in the src/pages directory, and the URL path will be based on the file name and its location within the directory structure. Dynamic routes allow you to capture URL parameters, and catch-all routes can match any path that doesn't match other defined routes.

010: scripts

- src/scripts

then create a script, and reference it within <body>,
at the end of the body:

<script>
    import "../scripts/script.js";
</script>


Better location is the frontmatter, but it can be anywhere in the file. 
The script will be bundled and included in the final output of the page.


.......................
011: layouts go in scr/layouts:
- src/layouts

then create a layout, and use it in the page:
- src/layouts/MainLayout.astro
at MainLayout.astro, add:
<slot />
to add the payload content to the layout, and

then at the page:
<MainLayout>
	<h2>Subtitle</h2>
</MainLayout>


The <slot /> allows you to inject (or “slot in”) child content written between opening and closing <Component></Component> tags to any Component.astro file.
